# MelCrochet — Plan 03: Public Site Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the public-facing MelCrochet Gifted Hands website — Home, Products, About, Blog, Contact — with the "editorial craft" design language from the spec, fully responsive, wired to the real Plan 02 API/query layer, with real WhatsApp ordering flows.

**Architecture:** Server Components read data directly via `lib/queries.ts` (Prisma, no HTTP hop). Public pages live under a new `app/(site)/` route group with a shared layout (Nav, Footer, floating WhatsApp button). A handful of small Client Components handle genuine interactivity (mobile nav drawer, testimonials carousel, the contact form). Design tokens from Plan 01 (`app/globals.css`) are extended, not replaced.

**Tech Stack:** Next.js 16 (Server Components, `next/image`, `next/font` already wired), Tailwind CSS v4, `react-markdown` (blog rendering), Vitest (existing `node`-environment suite stays untouched; a `jsdom` + React Testing Library environment is added per-file only where real DOM interaction is tested).

## Global Constraints

- **Route structure:** all public pages live under `app/(site)/` (a route group — doesn't affect URLs). `app/(site)/layout.tsx` wraps every public page with `Nav`, `Footer`, and the floating `WhatsAppButton`. The existing root `app/layout.tsx` (fonts, global metadata) is untouched. The current `app/page.tsx` (Plan 01's temporary colour-swatch check page) is **deleted** in Task 1 and replaced by `app/(site)/page.tsx`.
- **No product detail route.** Per the approved design, Products is a catalogue grid only (image, name, price/quote badge, WhatsApp order button) — no `/products/[slug]` page. Don't build one.
- **Images are all null right now.** Every seeded `Product`/`Testimonial`/`BlogPost` row has `imageUrl: null` (no photos uploaded yet — that happens later, through Plan 04's admin panel). **Every image-consuming component MUST render a graceful branded placeholder when the image field is null** — never a broken `<img>`, never blank space. Build and verify this placeholder path since it's the ONLY path realistically testable right now.
- **`next/image` + Cloudinary:** configure `next.config.ts`'s `images.remotePatterns` for `res.cloudinary.com` now, even though no real Cloudinary image exists yet to visually test against — this is required before Plan 04's uploads can render anywhere, and misconfiguring it now is cheap to catch, expensive to debug later.
- **WhatsApp:** canonical number `27670590600`. Generic link: `https://wa.me/27670590600?text=<url-encoded-message>`. Product-specific messages read `Hi MelCrochet! I'm interested in the {Product Name}`. Build this through one shared helper (`lib/whatsapp.ts`), never inline a `wa.me` URL by hand in a component.
- **Design language ("editorial craft" — see spec §5 for full detail):** Fraunces (`font-display`) headings, Inter (`font-sans`) body — already wired in Plan 01. Alternate `bg-ink text-cream` and `bg-cream text-ink` sections down each page for rhythm/contrast — don't default every section to the same background. Favor asymmetric, generous-whitespace editorial layouts over centered boxy templates. Use the shared `StitchDivider` component sparingly as a section accent, not on every section. Mobile-first: build the single-column layout first, then scale up with `sm:`/`md:`/`lg:` — never the reverse. WCAG AA contrast; every interactive element needs a visible focus state (Tailwind's `focus-visible:` utilities).
- **Data fetching:** Server Components import functions from `@/lib/queries` directly — no `fetch()`, no internal HTTP calls. Pages that read the database export `export const revalidate = 60;` (Next.js route-segment caching, which applies regardless of fetch-vs-direct-DB-read).
- **`CategoryFilter` is a plain (non-`"use client"`) component**, not a Client Component as the original spec text loosely suggested — it receives `activeSlug` as a prop (already resolved server-side from `searchParams`) and renders plain `<Link>` elements with `aria-current="page"` on the active one. This needs zero client JS, works with JS disabled, and is simpler to test. This is a deliberate, documented simplification of the spec's phrasing, not a deviation from its actual required behavior.
- **Testing:** the existing Vitest suite's global environment stays `"node"` — do not change it (Plan 02's 78 tests depend on it). Pure-logic pieces (`lib/whatsapp.ts`, `lib/youtube.ts`, `CategoryFilter`) are tested in the existing `node` environment (the latter via `react-dom/server`'s `renderToStaticMarkup`, no DOM needed). The Contact page's `EnquiryForm` is the one genuinely interactive Client Component in this plan needing real user-event simulation — Task 5 adds `jsdom` + React Testing Library scoped to that one test file via a `// @vitest-environment jsdom` docblock, not a global config change.
- **`fileParallelism: false` stays** (Plan 02 lesson — tests share one live database). Any test in this plan that writes to the DB (blog-post/enquiry browser-verification fixtures) must be cleaned up the same way established in Plan 02: create with an unmistakable `vitest-tmp-`/`Preview Test` marker, delete it before finishing the task.
- **Commits:** conventional format, no AI attribution lines.

```ts
const WHATSAPP_NUMBER = "27670590600";
const DEFAULT_MESSAGE =
  "Hi MelCrochet! I'd love to know more about your handmade creations.";

export function buildWhatsAppLink(message: string = DEFAULT_MESSAGE): string {
  return `https://wa.me/${WHATSAPP_NUMBER}?text=${encodeURIComponent(message)}`;
}

export function buildProductWhatsAppLink(productName: string): string {
  return buildWhatsAppLink(`Hi MelCrochet! I'm interested in the ${productName}`);
}
```

```tsx
"use client";

import { useState } from "react";
import { ChevronLeft, ChevronRight } from "lucide-react";
import ImagePlaceholder from "@/components/ImagePlaceholder";

type Testimonial = {
  id: string;
  customerName: string;
  quote: string;
  location: string | null;
  imageUrl: string | null;
};

export default function TestimonialsCarousel({
  testimonials,
}: {
  testimonials: Testimonial[];
}) {
  const [index, setIndex] = useState(0);

  if (testimonials.length === 0) {
    return (
      <p className="font-sans text-ink/60">
        Customer stories are on their way — check back soon.
      </p>
    );
  }

  const current = testimonials[index];
  const next = () => setIndex((i) => (i + 1) % testimonials.length);
  const prev = () => setIndex((i) => (i - 1 + testimonials.length) % testimonials.length);

  return (
    <div className="mx-auto max-w-2xl text-center">
      <div className="h-16 w-16 overflow-hidden rounded-full mx-auto">
        {current.imageUrl ? (
          // eslint-disable-next-line @next/next/no-img-element
          <img src={current.imageUrl} alt={current.customerName} className="h-full w-full object-cover" />
        ) : (
          <ImagePlaceholder className="h-full w-full" />
        )}
      </div>
      <p className="mt-6 font-display text-xl italic">&ldquo;{current.quote}&rdquo;</p>
      <p className="mt-3 font-sans text-sm font-semibold text-gold">
        {current.customerName}
        {current.location ? ` — ${current.location}` : ""}
      </p>

      {testimonials.length > 1 && (
        <div className="mt-6 flex items-center justify-center gap-4">
          <button
            type="button"
            onClick={prev}
            aria-label="Previous testimonial"
            className="flex h-9 w-9 items-center justify-center rounded-full border border-taupe/40 hover:border-gold"

            <ChevronLeft className="h-4 w-4" />

            aria-label="Next testimonial"
            className="flex h-9 w-9 items-center justify-center rounded-full border border-taupe/40 hover:border-gold"

            <ChevronRight className="h-4 w-4" />
```

```tsx
"use client";

import { useState, type FormEvent } from "react";

type Status = "idle" | "submitting" | "success" | "error";

export default function EnquiryForm() {
  const [status, setStatus] = useState<Status>("idle");
  const [errorMessage, setErrorMessage] = useState<string | null>(null);

  async function handleSubmit(event: FormEvent<HTMLFormElement>) {
    event.preventDefault();
    setStatus("submitting");
    setErrorMessage(null);

    const form = event.currentTarget;
    const formData = new FormData(form);
    const name = String(formData.get("name") ?? "").trim();
    const message = String(formData.get("message") ?? "").trim();

    if (!name || !message) {
      setStatus("error");
      setErrorMessage("Please fill in your name and message.");
      return;
    }

    try {
      const res = await fetch("/api/enquiries", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          name,
          email: String(formData.get("email") ?? ""),
          phone: String(formData.get("phone") ?? ""),
          message,
          website: String(formData.get("website") ?? ""),
        }),
      });

      if (!res.ok) {
        const body = await res.json().catch(() => ({ error: "Something went wrong." }));
        setStatus("error");
        setErrorMessage(body.error ?? "Something went wrong. Please try again.");
        return;
      }

      setStatus("success");
      form.reset();
    } catch {
      setStatus("error");
      setErrorMessage("Network error — please check your connection and try again.");
    }
  }

  if (status === "success") {
    return (
      <p role="status" className="font-sans text-ink">
        Thank you! Your message has been sent — we&apos;ll get back to you soon.
      </p>
    );
  }

  return (
    <form onSubmit={handleSubmit} className="flex flex-col gap-4" noValidate>
      {/* Honeypot — real visitors never see or reach this field */}
      <div className="absolute h-0 w-0 overflow-hidden" aria-hidden="true">
        <label htmlFor="website">Leave this field empty</label>
        <input id="website" name="website" type="text" tabIndex={-1} autoComplete="off" />
      </div>

      <div>
        <label htmlFor="name" className="font-sans text-sm font-medium">
          Name
        </label>
        <input
          id="name"
          name="name"
          type="text"
          required
          className="mt-1 w-full border border-taupe/40 bg-cream px-4 py-2 font-sans focus-visible:border-gold"
        />
      </div>

      <div>
        <label htmlFor="email" className="font-sans text-sm font-medium">
          Email (optional)
        </label>
        <input
          id="email"
          name="email"
          type="email"
          className="mt-1 w-full border border-taupe/40 bg-cream px-4 py-2 font-sans focus-visible:border-gold"
        />
      </div>

      <div>
        <label htmlFor="phone" className="font-sans text-sm font-medium">
          Phone (optional)
        </label>
        <input
          id="phone"
          name="phone"
          type="tel"
          className="mt-1 w-full border border-taupe/40 bg-cream px-4 py-2 font-sans focus-visible:border-gold"
        />
      </div>

      <div>
        <label htmlFor="message" className="font-sans text-sm font-medium">
          Message
        </label>
        <textarea
          id="message"
          name="message"
          rows={5}
          required
          className="mt-1 w-full border border-taupe/40 bg-cream px-4 py-2 font-sans focus-visible:border-gold"
        />
      </div>

      {status === "error" && errorMessage && (
        <p role="alert" className="font-sans text-sm text-red-700">
          {errorMessage}
        </p>
      )}

      <button
        type="submit"
        disabled={status === "submitting"}
```

- [ ] **Step 4: Implement `lib/youtube.ts`**

```ts
export function extractYouTubeId(url: string): string | null {
  if (!url) return null;

  try {
    const parsed = new URL(url);

    if (parsed.hostname === "youtu.be") {
      return parsed.pathname.slice(1) || null;
    }

    if (parsed.hostname.endsWith("youtube.com")) {
      const id = parsed.searchParams.get("v");
      return id || null;
    }

    return null;
  } catch {
    return null;
  }
}
```
